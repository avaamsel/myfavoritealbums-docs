Conceptual Overview

Audience

Developers:

This conceptual overview is intended for developers with very basic experience and understanding of the data analysis programming language, R, who have an interest in learning about and implementing new music-analysis features into MyFavoriteAlbums. Some familiarity using integrated development environments (IDEs) is assumed, as well as some familiarity with HTML web applications.

Data-Curious Music Lovers:

This conceptual overview is intended for Music Lovers with limited knowledge of R programming but have an interest in utilizing the MyFavoriteAlbums software to interact with their own music data. Music Lovers should have some comfort with dealing with files and the file system on their device.

What is MyFavoriteAlbums?

MyFavoriteAlbums is a streamlined web application that allows music lovers to interact with statistics about their music album preferences, especially over a range of years. MyFavoriteAlbums enables users to easily see patterns in their music and album preferences through a simple, yet intuitive interface and graphical displays.

Key Features:

  • Number One Albums: Users can specify a range of years and view a list of albums with the highest rating for each year in the range.
  • Top Albums By Year: Users can view a ranked list of all of their albums from a selected release year, with the number one album being the highest rated.
  • Artists’ Albums: Users can view a list of information for specified artist’s albums, including the total number of albums the user has ranked from the artist as well as the average rating of all of the artist’s albums that have been ranked by the user.
  • Favorite Artists: Users can view a list of artists with a specified minimum number of rated albums who have the highest average rating given by the user. Users also have the ability to exclude EPs and Live albums from the number of total albums artists have.
  • Artist Comparison: Users can compare the album ratings of two selected artists over a range of time.
  • Vinyl: Users can see their highest rated albums of a specified range that they do not own on vinyl.

Limitations:

  • MyFavoriteAlbums does not allow you to save the outputted statistical data and visualizations onto your device.
  • MyFavoriteAlbums requires users to manually upload their music data in CSV format within the software.

Software Architecture

Music Data:

MyFavoriteAlbum’s data is stored as a CSV (comma separated value) file on the application’s backend that is interacted with by the application’s R code to perform data analysis.

This CSV contains the following information formatted as columns:

  • Album year
  • Album ranking
  • Album name
  • Artist
  • Album rating on a scale of 1-10 (1 is the worst rating, 10 is the highest rating)
  • If the album is owned on vinyl - indicated by a "v"
  • If the album is an EP (extended play) - indicated by an "EP"
  • If the album is live - indicated by a "live"

R Code:

MyFavoriteAlbums is an R based web application. The application’s server builds the MyFavoriteAlbums with HTML and creates the individual analysis function tabs that can be interacted with by the user.

User Interface:

MyFavoriteAlbums’ user interface works closely with the application’s server as well as the R package, shiny’s function fluidpage to specify the layout of the web page, allowing for the tab-based structure of the application.

Tools and Environment

MyFavoriteAlbums requires developers to have R and RStudio installed onto their device. Similarly, users who wish to view their personal music data on MyFavoriteAlbums will also need to install R and RStudio, which can be downloaded by following the RStudio Desktop Downloading Instructions.

The packages shiny, dplyr, and ggplot2 are required for MyFavoriteAlbums to properly run.

  • shiny: turns the R code into an interactive web application.
  • dplyr: helps to manipulate the given music data for statistical analysis.
  • ggplot2: creates the visualization graphics for the given music rating data.

Data-Curious Music Lovers:

Users who wish to view their own music data will need to create a CSV file containing this information. This can be done using tools like Google Sheets, Excel, or TextEdit (macOS).

Users will also need to create a shinyapps.io account in order to deploy their own version of MyFavoriteAlbums and view their own music data on the application.

Task Topics for Developers:

Setting up the MyFavoriteAlbums Tech Stack:

MyFavoriteAlbums requires you to have R and RStudio downloaded onto your computer. If R and RStudio are not already downloaded onto your computer, download them by following the RStudio Desktop Downloading Instructions.

Cloning the MyFavoriteAlbums Repository:

  1. Access the RStudio terminal by clicking on the Terminal tab at the top of the bottom window.
  2. In your terminal, verify that you are currently located in your Desktop directory. Your directory path should look something like this:
    [device-name]:Desktop [user]$
  3. If you are not on your desktop, change your directory by entering the following command into your terminal:
    cd ~/Desktop
  4. Clone the MyFavoriteAlbums repository onto your computer.
    git clone https://github.com/UW-Example-Student/MyFavoriteAlbums.git
  5. Verify that the MyFavoriteAlbums repository was correctly cloned onto your desktop by entering the list command, ls, into your terminal:
    ls Desktop
    You should see the repository name, MyFavoriteAlbums, outputted in the list of your desktop contents if the repository has been cloned correctly.
  6. To download the packages required for MyFavoriteAlbums, run the following command in your RStudio console:
    install.packages(c("shiny", "dplyr", "tidyverse"))

Running MyFavoriteAlbums:

  1. Within the right Files sub-window, navigate to the app.R file.
  2. In the main coding window, click on the Run App button in the navigation bar to open a local version of the MyFavoriteAlbums web application.

Adding New Functions to MyFavoriteAlbums:

To add new functionality to MyFavoriteAlbums, you must have a basic understanding of R. For this tutorial, we will be creating a new function that will allow users to view a bar chart that displays the number of albums that were rated for each year in the data set.

  1. Create a new file to contain your new function by clicking the button and selecting R Script.
  2. In the "Create a New File” pop up box, name your new file chart.R and click OK. Make sure the name of your new file is in lowercase.
  3. Once you have named your file, navigate to app.R. Locate the line source("fav_bands.R") and paste the following code below it to define the UI layout and server functionality for chart.R:
    source("chart.R")
  4. Navigate to chart.R and fill out the file by creating the albums_by_year_chart function. Import the ggplot2 package to enable you to create a bar chart of the number of albums that were rated for each year in the data set. Paste the following code into your chart.R file: 
    library(ggplot2)

albums_by_year_chart <- function(album_data) {
  # Count albums per year
  year_counts <- as.data.frame(table(album_data$Year))
  colnames(year_counts) <- c("Year", "Count")
  year_counts$Year <- as.numeric(as.character(year_counts$Year))

  # Peak year
  peak_year <- year_counts$Year[which.max(year_counts$Count)]
  peak_count <- max(year_counts$Count)

  # Plot
  p <- ggplot(year_counts, aes(x = factor(Year), y = Count)) +
    geom_bar(stat = "identity", fill = "steelblue") +
    geom_bar(
      data = subset(year_counts, Year == peak_year),
      stat = "identity",
      fill = "red"
    ) +
    xlab("Year") +
    ylab("Number of Albums") +
    ggtitle(paste0(
      "Albums Rated by Year (Most Rated: ",
      peak_year, " with ", peak_count, ")"
    )) +
    theme_minimal() +
    theme(axis.text.x = element_text(angle = 90))

  print(p)
}

    When the albums_by_year_chart function is pasted into your chart.R file, it should look like this:

    The albums_by_year_chart function creates a bar chart of the number of albums that were rated for each year in the data set and stores it in the year_counts dataframe. The bar chart is created using ggplot2 which creates a bar for each year in the data set that has an album rated. The year with the most albums rated is highlighted in red.

  5. Navigate to app_server.R. To tell the server how to build the chart.R tab on the MyFavoriteAlbums web application, we will need to create a new server function. Paste the following code into your app_server.R file, below the code for the Vinyl tab: 
    output$year_chart <- renderPlot({
  albums_by_year_chart(album_data)
})

    When the output code is pasted into your app_server.R file, it should look like this:

    This code tell the server to render the bar chart used in chart.R. The albums_by_year_chart function is defined in chart.R and the data from our CSV file is passed to the function as album_data.

      When creating a new server function:

    • Use output to render any HTML contained in your file with renderUI. Output can also be used to render any tables or plots with renderPlot or renderTable.
    • Use observeEvent if there are any buttons needed for your new function to properly work. You will need to include what your button requires as an input, what it outputs, and what function it returns.
  6. Navigate to app_ui.R. To create a new tab for chart.R, make sure to reference which objects from the app_server.R file are being created by the new tab. Paste the following code into your app_ui.R file, below the tab-creation code for the Vinyl tab: 
    tabPanel(
  "Year Chart",
  plotOutput("year_chart")
)

    When the tabPanel code is pasted into your app_UI.R file, it should look like this:

    This code creates a new tab called Year Chart in MyFavoriteAlbums and displays the HTML output from the year_chart object in the app_server.R file.

      When creating a new UI tab:

    • Use htmlOutput to create HTML within your new tab.
    • Use selectInput if your new function utilizes a dropdown selector.
    • Use actionbutton if your new function utilizes a button.
    • Use tableOutput if your new function utilizes a table.
    • Use plotOutput if your new function utilizes a plot.
    • Use textOutput if your new function outputs any text.
  7. Ensure that all of your additions have been saved by RStudio before running the app so that chart.R appears on your local version of MyFavoriteAlbums.

If you would like to explore more dplyr and ggplot functionality, check out the dplyr documentation and the ggplot2 documentation to learn about more ways you can use these packages to perform statistical analysis on your album rating data.
